---
title: Заказы и реализации из лида
---

Лид можно «превратить» в документ, чтобы продолжить работу уже в учётном контуре и сохранить связь с исходным обращением.

В зависимости от настройки доступны:

- создание **[заказа](../sales/orders.md)** из лида;
- создание **[реализации](../sales/invoices.md)** из лида;
- просмотр уже созданных связанных документов в карточке лида.

## Подготовка (один раз)

Доступность кнопок создания документов зависит от параметров в разделе **«Настройка → Настройки»**:

- **«Тип заказа»** — определяет, какой тип будет подставлен при создании заказа;
- **«Тип реализации»** — определяет, какой тип будет подставлен при создании реализации.

Если параметр не задан, соответствующая кнопка в карточке лида не отображается.

Подробности см. в документе [Настройки и справочники](settings.md).

## Когда стоит создавать документы из лида

Создание [заказа](../sales/orders.md) или [реализации](../sales/invoices.md) удобно, когда:

- потребность клиента подтверждена;
- понятны ключевые условия (что продаём, кому, куда);
- требуется фиксировать дальнейшие действия уже в документах.

Если запрос пока «сырой», лучше сначала уточнить детали и вести лид по статусам.

## Создание заказа из лида

### Как создать

1. Откройте карточку лида.
2. Нажмите **«Создать Заказ»**.
3. Система создаст новый заказ и откроет его для дальнейшего заполнения.

Перед созданием рекомендуется проверить в лиде:

- заполнен ли [контрагент](../masterdata/partners.md);
- корректны ли телефон и электронная почта;
- заполнены ли адресные поля (если доставка важна).

### Что заполняется автоматически

Как правило, при создании автоматически переносятся:

- ссылка на исходный лид;
- **[контрагент](../masterdata/partners.md)** из лида;
- **тип** = значение параметра **«Тип заказа»**;
- **адрес доставки** — формируется из адресных данных лида (если они заполнены).

Если часть данных не заполнена в лиде (например, нет [контрагента](../masterdata/partners.md)), в заказе её потребуется указать вручную.

### Где посмотреть связанные заказы

В карточке лида отображается блок **«Заказы»**:

- показываются только те заказы, которые связаны с текущим лидом;
- в списке обычно видны номер, дата/время, статус, тип, сумма и другие ключевые поля;
- из списка можно открыть заказ на редактирование.

Также связь обычно видна и в самом заказе (поле «Лид» в документе), поэтому найти исходный лид можно и «из документа».

## Создание реализации из лида

### Как создать

1. Откройте карточку лида.
2. Нажмите **«Создать реализацию»**.
3. Система создаст новый документ реализации и откроет его.

Рекомендация: создавайте реализацию из лида, когда уже понятно, что [отгрузка](../sales/shipments.md)/оказание услуги будет оформляться в ближайшее время.

Рекомендуемая статья по документу: [Реализации по заказам](../sales/invoices.md).

### Что заполняется автоматически

Как правило, при создании автоматически переносятся:

- ссылка на исходный лид;
- **[контрагент](../masterdata/partners.md)** из лида;
- **тип** = значение параметра **«Тип реализации»**;
- **адрес доставки** — формируется из адресных данных лида (если они заполнены).

### Где посмотреть связанные реализации

В карточке лида отображается блок **«Реализации»**:

- показываются документы реализации, созданные из текущего лида;
- из списка можно открыть документ на редактирование.

## Что делать, если документ создан ошибочно

1. Откройте созданный документ.
2. Если документ пустой и не использовался — его можно удалить (при наличии прав).
3. Если документ уже участвовал в процессе, согласуйте действия с ответственными пользователями (например, отмена/закрытие документа в рамках правил учёта).
4. Вернитесь в лид и исправьте исходные данные ([контрагент](../masterdata/partners.md), адрес, тип), чтобы повторное создание было корректным.

## Типовые ситуации

- **Кнопки создания нет** — не задан «Тип заказа» или «Тип реализации» в настройках, либо недостаточно прав.
- **Неправильный [контрагент](../masterdata/partners.md)** — проверьте поле «[Контрагент](../masterdata/partners.md)» в лиде перед созданием документа.
- **Адрес не заполнился** — заполните адресные поля в лиде и повторите создание (или заполните адрес в документе вручную).
