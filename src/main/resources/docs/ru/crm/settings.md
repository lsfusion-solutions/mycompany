## Где находится

Откройте раздел **«Настройка»**, затем **«Настройки»**.

Здесь собраны параметры и справочники, которые определяют, как пользователи ведут лиды: какие бывают статусы, типы, приоритеты, ярлыки, а также как фиксируется поражение.

## Параметры

### Тип заказа

Параметр **«Тип заказа»** определяет, какой тип будет подставляться при создании [заказа](../sales/orders.md) из лида.

Если значение не задано, кнопка **«Создать Заказ»** в карточке лида не отображается.

### Тип реализации

Параметр **«Тип реализации»** определяет, какой тип будет подставляться при создании [реализации](../sales/invoices.md) из лида.

Если значение не задано, кнопка **«Создать реализацию»** в карточке лида не отображается.

## Справочники

### Статусы лида

Статус определяет этап работы с лидом.

Важные поля:

- **Наименование** — как статус будет показан пользователю;
- **Идентификатор** — внутренний код (используется в настройках и интеграциях);
- **Порядок сортировки** — порядок отображения в списках и на доске по лидам;
- **Закрыта** — признак закрытого статуса (закрытые статусы не показываются на доске);
- **Поражение** — признак статуса, который должен устанавливаться при фиксации поражения.

Рекомендации:

1. Заведите минимум 3–5 статусов, чтобы воронка была понятной.
2. Для закрывающих статусов обязательно отметьте **«Закрыта»**.
3. Выберите один статус и отметьте его как **«Поражение»** — он будет устанавливаться при закрытии лида через поражение.
4. Проставьте **порядок сортировки** так, чтобы колонки на доске шли слева направо по логике процесса.

### Типы лида

Тип лида — это классификация обращения. Тип влияет на доступные статусы.

Как работает ограничение статусов:

- для типа можно явно указать, какие статусы допустимы;
- при сохранении лида система проверяет, что выбранный статус разрешён для типа.

Рекомендация: если у вас несколько разных воронок (например, «опт», «розница», «сервис»), создайте отдельные типы и задайте допустимые статусы для каждого.

### Приоритеты лида

Приоритет помогает выделять важные лиды.

Обычно у приоритета есть:

- **наименование**;
- **цвет**.

Цвет может отображаться в списке и в карточке лида.

### Ярлыки лида

Ярлыки — метки, которые можно назначать лидам для удобной группировки.

В справочнике ярлыков обычно задаются:

- **Наименование**;
- **Идентификатор**;
- **цвет**.

Рекомендация: используйте ярлыки для сквозных признаков (источник, сегмент, тип запроса), не дублируя ими статусы.

### Причины поражений

Причины поражений используются при закрытии лида через действие **«Поражение»**.

Как это выглядит для пользователя:

1. пользователь нажимает **«Поражение»** в карточке лида;
2. выбирает причину поражения;
3. причина сохраняется в лиде, и лид переводится в статус, помеченный в справочнике статусов как **«Поражение»**.

Рекомендация: заведите причины так, чтобы по ним можно было делать выводы (например, «слишком дорого», «выбрали конкурента», «нет бюджета», «не дозвонились», «не актуально»).

## Рекомендуемый порядок первичной настройки

1. Создайте статусы лида и настройте порядок сортировки.
2. Отметьте закрывающие статусы и выберите статус поражения.
3. Создайте типы лида и задайте допустимые статусы.
4. Настройте приоритеты и ярлыки (при необходимости).
5. Заполните причины поражений.
6. Задайте параметры «Тип заказа» и «Тип реализации», если планируете создавать документы из лида.
