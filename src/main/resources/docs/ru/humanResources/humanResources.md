## Быстрый старт

Типовой сценарий «получить отклик кандидата → назначить собеседование → принять решение»:

1. Откройте **«Кадры» → «Операции» → «Анкеты»**.
2. Создайте анкету и заполните основные реквизиты (тема/описание, должность, подразделение, контактные данные кандидата).
3. При необходимости прикрепите файлы (например, резюме).
4. Назначьте собеседование и выберите участников.
5. По результатам собеседования:
   - выполните действие **«Нанять»**, чтобы оформить сотрудника;
   - или выполните действие **«Отказать»** и укажите причину отказа.

Типовой сценарий «зафиксировать рабочее время»:

1. Откройте **«Кадры» → «Операции» → «Регистрация времени»**.
2. Зарегистрируйте **«Вход»** и **«Выход»** одним из способов:
   - через мобильную регистрацию времени (с геолокацией, если используется);
   - через **киоск регистрации времени** по бейджу.

Типовой сценарий «сформировать зарплату и подготовить выплату»:

1. Откройте **«Кадры» → «Операции» → «Зарплатные ведомости»**.
2. Создайте ведомость, укажите организацию и период.
3. Выполните действие **«Сформировать»** — система создаст расчётные листы по сотрудникам.
4. Проверьте расчётные листы и итог **«К выдаче»**.
5. Если в организации используется регистрация выплат в системе — зарегистрируйте выплату по расчётному листу.

## Навигация

Раздел **«Кадры»** обычно содержит группы:

- **Операции** — ежедневная работа (анкеты, собеседования, регистрация времени, расчётные листы и зарплатные ведомости).
- **Процессы** — представления для контроля и обработки (например, сводные списки, обработка входящих сообщений, если подключено).
- **Настройка** — справочники и параметры раздела.

Набор пунктов меню и доступность действий зависят от настроек и прав пользователя.

## Роли пользователей и права

Точный набор прав зависит от настроек вашей организации. Типовая логика распределения ответственности:

- **Рекрутер** — ведёт анкеты, назначает собеседования, фиксирует результаты, готовит решения.
- **Участник собеседования** — участвует в собеседованиях и вносит результаты (если разрешено).
- **Кадровый специалист** — оформляет приём сотрудников, ведёт кадровые данные.
- **Сотрудник** — фиксирует рабочее время (мобильная регистрация или киоск), просматривает свои записи (если разрешено).
- **Бухгалтер/расчётчик** — формирует зарплатные ведомости, проверяет расчётные листы и регистрирует выплаты (если используется).

Если некоторые действия недоступны (например, «Нанять», «Отказать», «Сформировать» или регистрация выплат), это обычно связано с ограничениями прав или с настройками раздела.

## Термины

#### Анкета

Карточка кандидата: содержит сведения о кандидате, должности, контактные данные, прикреплённые файлы и историю работы по подбору.

#### Собеседование

Запись о проведении собеседования с кандидатом: фиксирует участников и сводку/результат.

#### Рекрутер

Ответственный за ведение подбора по анкете.

#### Участник собеседования

Пользователь, который участвует в собеседовании кандидата.

#### Регистрация времени

Фиксация факта входа/выхода сотрудника. В зависимости от настроек может записываться геолокация и/или фотография.

#### Киоск регистрации времени

Экран/устройство для быстрого входа/выхода, когда сотрудник отмечается по бейджу.

#### Зарплатная ведомость

Документ для группового формирования расчётных листов сотрудников за период.

#### Расчётный лист

Документ расчёта начислений и удержаний сотрудника за период с итогом **«К выдаче»**.
