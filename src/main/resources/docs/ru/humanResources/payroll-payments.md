---
title: Выплата зарплаты и контроль выплат
---

В некоторых организациях выплаты фиксируются в системе: по расчётному листу можно зарегистрировать платёж и контролировать, сколько уже выплачено и сколько осталось.

Если в вашем интерфейсе нет действий для регистрации выплат, значит функция может быть **не включена** настройками или недоступна по правам.

## Зарегистрировать выплату по расчётному листу

Рекомендуемый порядок работы:

1. Откройте расчётный лист сотрудника.
2. Проверьте итог **«К выдаче»**.
3. Выполните действие **«Оплатить»** — система открывает документ исходящего платежа, заполненный непогашенной суммой.
4. Проверьте сумму платежа; для частичной выплаты уменьшите её.
5. Сохраните исходящий платёж.

С расчётным листом может быть связано несколько платежей — например, аванс и окончательный расчёт. Связанные с расчётным листом платежи показываются в блоке **«Платежи»** карточки расчётного листа. После регистрации платёж учитывается в контроле выплат ниже.

## Контроль выплат

В списке расчётных листов (**«Кадры» → «Операции» → «Расчётные листы»**) по каждому расчётному листу видна сумма **«Оплачено»** и доступен фильтр **«Не оплачено»**.

На вкладке **«Итого»** списка отображаются сводные показатели за диапазон дат:

- **«К выдаче»** — сумма итогов «К выдаче» по расчётным листам диапазона;
- **«Оплачено»** — сумма зарегистрированных выплат за диапазон;
- **«Осталось»** — «К выдаче» за вычетом «Оплачено».

Кнопка **«Платеж»** (клавиша `Insert`) списка платежей на этой вкладке создаёт исходящий платёж по выбранному сотруднику: сумма заполняется значением **«Осталось»** за выбранный диапазон, но платёж изначально распределяется по **всем** расчётным листам сотрудника без учёта диапазона — проверьте распределение перед сохранением.

#### На что обратить внимание

- Если выплата по расчётному листу зарегистрирована частично, **«Осталось»** не станет нулевым.
- Если по сотруднику есть несколько расчётных листов в диапазоне, контроль суммирует их все.
- Даты сравниваются по-разному: **«К выдаче»** учитывает расчётные листы, у которых в диапазон попадает **начало периода**, а **«Оплачено»** — платежи по **дате платежа**, включая все платежи данного типа по сотруднику, даже не связанные с показанными расчётными листами.
