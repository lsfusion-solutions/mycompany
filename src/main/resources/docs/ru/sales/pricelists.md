---
title: Прайс-листы и типы цен
---

Прайс-листы используются для хранения и применения цен в заказах покупателя.

![Список прайс-листов](images/pricelists-list.png)

## Где находится

- **Прайс-листы** — **«Продажи» → «Операции» → «Прайс-листы»**;
- **Типы цен** — **«Продажи» → «Настройка» → «Типы цен»**;
- **Типы прайс-листов** — на форме **«Продажи» → «Настройка» → «Настройки»**.

## Типы цен

**Тип цены** — это именованная ценовая шкала. У каждого типа цены есть валюта и признак «цена включает налоги», а также могут быть заданы наценки по умолчанию для категорий товаров. Типы цен используются как ценовые колонки прайс-листа и для выбора цены, подставляемой в строку заказа.

## Прайс-лист

Карточка прайс-листа содержит:

- **номер** и необязательное **примечание**;
- **тип прайс-листа** — категорию, по которой группируются прайс-листы (например, «Оптовый», «Розничный», «Акционный»);
- **период действия** — дату начала и окончания;
- **типы цен**, колонки которых редактируются в этом прайс-листе;
- список **номенклатуры и цен** — по одной ценовой колонке на каждый выбранный тип цены.

## Статусы прайс-листа

Прайс-лист обычно проходит два статуса:

1. **Черновик** — значения цен можно изменять; прайс-лист пока не используется в подстановках цен.
2. **Готов** — прайс-лист принят в работу; значения становятся источником цен на свою дату действия.

Перевод в статус «Готов» выполняется действием **«Отметить как готовый»** в карточке прайс-листа.

## Редактирование прайс-листа

В карточке прайс-листа есть инструменты для массового заполнения цен:

- **дерево категорий и поиск номенклатуры** для добавления нужных позиций в строки;
- действие **«Изменить цены»**, которое пересчитывает все редактируемые цены прайс-листа одним из трёх способов — задать наценки по предыдущим ценам, задать наценки по типам цен или взять цены из предыдущего прайс-листа (при необходимости скорректировав на процент);
- действие **«Копировать»**, которое создаёт новый прайс-лист на основе текущего.

Каждую ценовую колонку также можно сравнить с текущей действующей ценой номенклатуры.

## Использование в заказе

При добавлении строки заказа система подставляет цену по **номенклатуре**, **типу цены** и **дате/времени заказа**. Сам тип цены определяется по контрагенту или по типу заказа.

Цена берётся из последнего прайс-листа в статусе **«Готов»**, период действия которого покрывает дату документа. Если значение в прайс-листах не найдено, система использует собственную цену продажи номенклатуры.
