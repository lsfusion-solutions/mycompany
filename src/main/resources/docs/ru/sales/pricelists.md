---
title: Прайс-листы и виды цены
---

Прайс-листы используются для хранения и применения цен в заказах покупателя и реализациях.

![Список прайс-листов](images/pricelists-list.png)

## Где находится

- **Прайс-листы** — **«Продажи» → «Операции» → «Прайс-листы»**;
- **Виды цены** — **«Продажи» → «Настройка» → «Виды цены»**;
- **Типы прайс-листов** — на форме **«Продажи» → «Настройка» → «Настройки»**.

## Виды цены

**Вид цены** — это именованная ценовая шкала. У каждого вида цены есть валюта и флаг **«С учетом налогов»** (включают ли его цены налоги), а также могут быть заданы надбавки по умолчанию для категорий товаров. Виды цены используются как ценовые колонки прайс-листа и для выбора цены, подставляемой в строку заказа.

## Прайс-лист

Карточка прайс-листа содержит:

- **номер** и необязательное **примечание**;
- **тип прайс-листа** — категорию, по которой группируются прайс-листы (например, «Оптовый», «Розничный», «Акционный»);
- **период действия** — дату начала и окончания;
- **виды цены**, колонки которых редактируются в этом прайс-листе;
- список **номенклатуры и цен** — по одной ценовой колонке на каждый выбранный вид цены.

Тип прайс-листа не только группирует прайс-листы, но и настраивает их поведение: нумератор для нумерации, какие колонки видов цены доступны для редактирования, а также флаги **«Показывать себестоимость»**, **«Показывать надбавки»** и **«Показывать действующие цены»**. Для каждого вида цены поле **«Считать надбавку от»** задаёт базовый вид цены, от которого рассчитывается надбавка.

## Статусы прайс-листа

Прайс-лист обычно проходит два статуса:

1. **Черновик** — значения цен можно изменять; прайс-лист пока не используется в подстановках цен.
2. **Выполнен** — прайс-лист принят в работу; значения становятся источником цен на свою дату действия.

Перевод в статус «Выполнен» выполняется действием **«Провести»** в карточке прайс-листа.

## Редактирование прайс-листа

В карточке прайс-листа есть инструменты для массового заполнения цен:

- **дерево категорий и поиск номенклатуры** для добавления нужных позиций в строки;
- действие **«Изменить цены»**, которое пересчитывает все редактируемые цены прайс-листа одним из трёх способов — **«Установить надбавки из предыдущих прайсов»**, **«Установить надбавки из видов цен»** или **«Установить цены из предыдущих прайсов»** (при необходимости скорректировав на процент в поле **«Изменить на, %»**);
- действие **«Копировать»**, которое создаёт новый прайс-лист на основе текущего.

Каждую ценовую колонку также можно сравнить с текущей действующей ценой номенклатуры.

Прайс-лист продажи можно также создать на основе прайс-листа поставщика (закупки) — действием **«Расценить»** в карточке прайс-листа закупки; используемый тип прайс-листа задаётся настройкой **«Тип прайс-листа на продажу»**.

Есть печатные формы: прайс-лист можно распечатать из карточки, а также распечатать действующие цены по категориям и видам цены.

## Использование в заказе

При добавлении строки заказа система подставляет цену по **номенклатуре**, **виду цены** и **дате/времени заказа**. Сам вид цены определяется по контрагенту или по типу заказа.

Цена берётся из последнего прайс-листа в статусе **«Выполнен»**, период действия которого покрывает дату документа. Если значение в прайс-листах не найдено, система использует собственную цену продажи номенклатуры. Аналогично цены подставляются в реализациях.

В карточке номенклатуры на вкладке **«Цены»** доступна история цен по каждому виду цены — с указанием прайс-листов, которыми установлена каждая цена.
