title: "Себестоимость: как рассчитывается"

Этот раздел описывает, как система рассчитывает себестоимость изделия при выполнении [производственного заказа](orders.md).

Описание составлено по логике, реализованной в модуле расчёта себестоимости [производственных заказов](orders.md).

## Общая идея

Себестоимость считается **по факту**:

1. Сначала в [производственном заказе](orders.md) фиксируется **фактическое потребление материалов**.
2. По факту потребления рассчитывается **стоимость списания материалов**.
3. Сумма списаний (плюс дополнительные составляющие себестоимости, если они есть) образует **себестоимость производственного заказа**.
4. Себестоимость заказа **распределяется на выпуск** (на строки выпуска) по долям стоимости.

## Что участвует в расчёте

### 1) Фактическое потребление материалов

В [производственном заказе](orders.md) есть строки материалов. Для каждой строки фиксируется фактическое количество **«Израсходовано»**.

В расчёт себестоимости попадают только строки, где:

- фактическое потребление заполнено;
- номенклатура является товарной/материальной (участвует в учёте запасов).

Сумма списания материалов по строке отображается в её колонке **«Себестоимость»**.

### 2) Место хранения материалов и дата исполнения

Стоимость потребления определяется с учётом:

- места хранения материалов (**«Списать из»** — место хранения, из которого идёт списание);
- **«Даты исполнения»** [производственного заказа](orders.md).

Практический смысл:

- если в разных местах хранения разная себестоимость, списание оценивается по месту хранения материалов;
- **«Дата исполнения»** фиксирует «момент оценки», то есть по какой дате берётся стоимость.

### 3) Дополнительные составляющие себестоимости (типы себестоимости)

Помимо списания материалов себестоимость производственного заказа может включать дополнительные составляющие. Внутри системы они унифицированы как **типы себестоимости**: каждый тип — отдельный источник, накапливающий свою сумму на заказе, и конфигурации могут добавлять собственные типы. Это **вычисляемые итоги**, отображаемые в заказе только для чтения, — их не вводят напрямую. Стандартные типы:

- **«Дополнительная себестоимость»** — накапливает стоимость документов [списания](scrap.md), привязанных к заказу;
- **«Себестоимость труда»** — накапливает суммы оплаты труда из [проектных](../projectManagement/projectManagement.md) отметок времени, привязанных к заказу (если используется контур управления проектами);
- **«Себестоимость услуг»** — накапливает затраты из [поступлений](../invoicing/bills.md) поставщиков: строка услуги поступления, распределённая на заказ, добавляет свою долю.

Каждая составляющая появляется только при наличии соответствующих исходных данных в вашей конфигурации.

> Себестоимость услуг из поступлений использует стандартный механизм распределения затрат (см. [Распределение на себестоимость](../invoicing/bill-cost.md) в «Расчётах»): в блоке распределения затрат поступления есть вкладка **«Производственные заказы»**, где строку услуги поступления можно пометить флагом **«Распределить»** на открытые производственные заказы; базой распределения могут быть сумма, себестоимость, цена продажи, вес, объём или количество заказа, а доля **«Распределено»**, попавшая на конкретный заказ, становится его себестоимостью услуг.

## Формулы

### Себестоимость материалов

Себестоимость материалов по заказу (колонка **«Себестоимость»**) — это сумма стоимостей списаний по всем строкам материалов.

Иными словами:

- для каждой строки материалов рассчитывается сумма списания;
- затем эти суммы складываются в себестоимость заказа.

### Общая себестоимость производственного заказа

**«Общая себестоимость»** заказа складывается из:

1. себестоимости материалов (**«Себестоимость»**);
2. плюс все составляющие по типам себестоимости: **«Дополнительная себестоимость»**, **«Себестоимость труда»**, **«Себестоимость услуг»** (и любые типы, добавленные в вашей конфигурации).

**Себестоимость материалов** и построчные суммы по выпуску формируются **при переводе заказа в статус «Выполнен»** — записи регистра себестоимости, оценивающие списание материалов, создаются только для завершённых (в статусе «Выполнен», не «Отменен») заказов, поэтому себестоимость материалов остаётся пустой, пока заказ находится в статусе «Черновик» / «В ожидании» / «В работе» / «В процессе». Дополнительные составляющие ведут себя иначе: они накапливаются из привязанных записей (документов списания, отметок времени, распределений по поступлениям) независимо от статуса заказа, поэтому они — а значит, и общая себестоимость — могут быть ненулевыми ещё до перевода заказа в «Выполнен». Перевод заказа в «Выполнен» также фиксирует **«Дату исполнения»**, которая определяет момент оценки для списания материалов.

## Как себестоимость распределяется на выпуск

В заказе есть строки выпуска (какие изделия/полуфабрикаты произведены). Себестоимость заказа распределяется между строками выпуска.

### Доля стоимости

У строки выпуска может быть заполнена **«Доля стоимости»** (коэффициент распределения).

Расчёт доли строки:

- если доли стоимости заполнены хотя бы у одной строки, то доля строки = коэффициент строки / сумма коэффициентов по заказу;
- если доли стоимости не заполнены, вся себестоимость относится на основное изделие заказа (на строку выпуска, где номенклатура совпадает с номенклатурой заказа). Для заказов на [разборку](unbuild.md) такой строки нет, поэтому без долей стоимости себестоимость на компоненты не распределяется вовсе — заполните доли (они переносятся из компонентов спецификации).

### Стоимость строки выпуска

Сумма, которая будет отнесена на конкретную строку выпуска:

**стоимость строки выпуска = общая себестоимость заказа × доля строки**

### Себестоимость единицы

Чтобы получить себестоимость единицы по строке выпуска:

**себестоимость единицы = стоимость строки выпуска / фактически произведено по строке**

Если фактически произведено равно нулю, себестоимость единицы рассчитать нельзя — сначала зафиксируйте выпуск.

## Где посмотреть себестоимость

Обычно себестоимость видна:

- в списке [производственных заказов](orders.md) — колонки только для чтения **«Себестоимость»**, **«Дополнительная себестоимость»**, **«Общая себестоимость»**, а также **«Себестоимость труда»** и **«Себестоимость услуг»**, если используются соответствующие контуры;
- на вкладке **«Итого»** списка (по выделенным заказам) — агрегированная **«Себестоимость»** материалов в разрезе номенклатуры;
- в [отчётах по производству](reports.md) — те же показатели себестоимости в сводном отчёте.

## Типовые ситуации и проверки

#### Общая себестоимость равна нулю

Проверьте:

- заполнено ли «Израсходовано» по строкам материалов;
- выбрано ли место хранения материалов («Списать из»);
- находится ли заказ в статусе **«Выполнен»** (задана **«Дата исполнения»**) — себестоимость материалов оценивается только при завершении заказа.

#### Себестоимость «вся ушла» на одну строку выпуска

Это нормально, если доли стоимости не заполнены. Тогда система относит всю сумму на основное изделие.

#### Себестоимость распределилась «не так, как ожидается»

Проверьте:

- доли стоимости в строках выпуска;
- фактически произведённые количества;
- не добавлены ли дополнительная себестоимость, себестоимость труда или услуг.
