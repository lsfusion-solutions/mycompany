Этот раздел описывает, как в системе рассчитывается себестоимость изделия при выполнении [производственного заказа](orders.md).

Описание составлено по логике, реализованной в модуле расчёта себестоимости [производственных заказов](orders.md).

## Общая идея

Себестоимость считается **по факту**:

1. Сначала фиксируется **фактическое потребление материалов** по [производственному заказу](orders.md).
2. По факту потребления формируется **стоимость списания материалов**.
3. Сумма списаний (плюс дополнительные и трудовые затраты, если они заданы) образует **себестоимость производственного заказа**.
4. Себестоимость заказа **распределяется на выпуск** (на строки выпуска) по коэффициентам распределения.

## Что участвует в расчёте

### 1) Фактическое потребление материалов

В [производственном заказе](orders.md) есть строки материалов. Для каждой строки фиксируется фактическое количество **«Потреблено»**.

В расчёт себестоимости попадают только строки, где:

- фактическое потребление заполнено;
- номенклатура является материальной (участвует в учёте запасов).

### 2) Место хранения материалов и дата выполнения

Стоимость потребления определяется с учётом:

- **места хранения материалов** (место хранения, из которого идёт списание);
- **даты выполнения** [производственного заказа](orders.md).

Практический смысл:

- если в разных местах хранения разная себестоимость, списание будет оценено по месту хранения материалов;
- дата выполнения фиксирует «момент оценки», то есть по какой дате берётся стоимость.

### 3) Дополнительные, трудовые и сервисные затраты

Помимо списания материалов себестоимость производственного заказа может включать ещё три составляющих. Это **вычисляемые итоги**, отображаемые в заказе только для просмотра, — их не вводят в заказ напрямую. Каждая складывается из своего источника:

- **Дополнительные затраты** — накапливают стоимость документов [списания](scrap.md), привязанных к заказу;
- **Трудовые затраты** — накапливают суммы оплаты труда из [проектных](../projectManagement/projectManagement.md) отметок времени, привязанных к заказу (если используется контур «Проекты»);
- **Сервисные затраты** — накапливают затраты из [поступлений](../invoicing/bills.md) поставщиков: сервисная строка поступления, распределённая на заказ, добавляет свою долю.

Каждая составляющая появляется только при наличии соответствующих исходных данных в вашей конфигурации.

> Сервисные затраты из поступлений используют стандартный механизм распределения затрат (см. [Распределение затрат](../invoicing/bill-cost.md) в «Расчётах»): сервисную строку поступления можно распределить на производственные заказы, и доля, попавшая на конкретный заказ, становится его сервисными затратами.

## Формулы

### Себестоимость материалов

Себестоимость материалов по заказу — это сумма стоимостей списаний по всем строкам материалов.

Иными словами:

- для каждой строки материалов считается сумма списания;
- затем суммы складываются в себестоимость заказа.

### Итоговая себестоимость производственного заказа

**Итоговая себестоимость заказа** складывается из:

1. себестоимости материалов;
2. дополнительных затрат;
3. трудовых затрат;
4. сервисных затрат (из привязанных [поступлений](../invoicing/bills.md)).

**Себестоимость материалов** и построчные суммы по выпуску формируются **при переводе заказа в «Выполнен»** — записи регистра себестоимости, оценивающие списание материалов, создаются только для завершённых (в статусе «Выполнен», не «Отменён») заказов, поэтому себестоимость материалов остаётся пустой, пока заказ находится в «Черновике» / «Ожидании» / «Готов» / «В работе». Дополнительные, трудовые и сервисные затраты ведут себя иначе: они накапливаются из привязанных документов (списаний, отметок времени, распределений по поступлениям) независимо от статуса заказа, поэтому они — а значит, и итоговая себестоимость — могут быть ненулевыми ещё до перевода заказа в «Выполнен». Перевод заказа в «Выполнен» также фиксирует **дату выполнения**, которая определяет момент оценки для списания материалов.

## Как себестоимость распределяется на выпуск

В заказе есть строки выпуска (какие изделия/полуфабрикаты произведены). Себестоимость заказа распределяется между строками выпуска.

### Коэффициент распределения

Для строки выпуска может быть заполнен коэффициент **«Коэффициент распределения себестоимости»**.

Расчёт доли строки:

- если коэффициенты заполнены хотя бы у одной строки, то доля строки = коэффициент строки / сумма коэффициентов по заказу;
- если коэффициенты не заполнены, то вся себестоимость относится на основное изделие заказа (на строку выпуска, где номенклатура совпадает с номенклатурой заказа).

### Стоимость выпуска

Стоимость, которая будет отнесена на конкретную строку выпуска:

**стоимость строки выпуска = итоговая себестоимость заказа × доля строки**

### Себестоимость единицы

Чтобы получить себестоимость единицы по строке выпуска:

**себестоимость единицы = стоимость строки выпуска / фактически произведено по строке**

Если фактически произведено равно нулю, себестоимость единицы рассчитать нельзя — сначала зафиксируйте выпуск.

## Где посмотреть себестоимость

Обычно себестоимость видна:

- в списке [производственных заказов](orders.md) — поля «Себестоимость», «Дополнительные затраты», «Трудовые затраты», «Итоговая себестоимость»;
- в [отчетах по производству](reports.md) (например, в отчете по заказам, если он включён в разделе отчетности).

## Типовые ситуации и проверки

#### Итоговая себестоимость равна нулю

Проверьте:

- заполнено ли «Потреблено» по строкам материалов;
- выбрано ли место хранения материалов;
- задана ли дата выполнения (обычно появляется при завершении заказа).

#### Себестоимость «вся ушла» на одну строку выпуска

Это нормально, если коэффициенты распределения не заполнены. Тогда система относит всю сумму на основное изделие.

#### Себестоимость распределилась «не так, как ожидается»

Проверьте:

- коэффициенты распределения по строкам выпуска;
- фактически произведенные количества;
- не добавлены ли дополнительные/трудовые затраты.
