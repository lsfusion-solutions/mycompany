Этот раздел описывает, как использовать **Рабочие центры** и **Производственные задания** для планирования и управления производственными операциями.

## Рабочие центры

**Рабочий центр** — это производственная единица (станок, группа станков или определенный участок), где выполняются производственные операции.

### Управление рабочими центрами

Для управления рабочими центрами перейдите в **Производство** → **Настройка** → **Рабочие центры**.

Для каждого рабочего центра можно указать:
- **Наименование** — описательное название рабочего центра (например, "Линия сборки А", "Станок ЧПУ 1").
- **ID** — уникальный внутренний код рабочего центра.
- **Описание** — подробная информация о возможностях или расположении центра.

### Практическое использование
Рабочие центры используются для группировки производственных заданий и анализа общей производственной нагрузки. Определив отдельные рабочие центры, вы сможете выявлять "узкие места" и балансировать нагрузку на ваше предприятие.

## Производственные задания

**Производственное задание** представляет собой конкретную операцию или задачу, выполняемую в рабочем центре в рамках [производственного заказа](orders.md). В то время как производственный заказ определяет *что* производится, производственные задания определяют *как* и *где* выполняется работа.

### Создание производственных заданий

Производственные задания обычно создаются в контексте **Производственного заказа**. Вы можете разбить один производственный процесс на несколько последовательных или параллельных операций.

#### Шаги по добавлению производственного задания:
1. Откройте карточку **Производственного заказа**.
2. Перейдите на вкладку **Производственные задания**.
3. Нажмите **Добавить** (Создать), чтобы создать новую строку.
4. Заполните следующие данные:
   - **Наименование** — название операции (например, "Резка", "Сварка").
   - **Рабочий центр** — выберите подразделение, где будет проходить эта операция (обязательно).
   - **Дата начала** — день, когда запланировано начало операции (по умолчанию — плановая дата производственного заказа).
   - **Время начала** — конкретное время суток, когда должна начаться операция.
   - **Длительность** — расчетное время, необходимое для выполнения операции (в часах).

### Управление всеми производственными заданиями

Чтобы увидеть глобальный список всех производственных задач, перейдите в **Производство** → **Операции** → **Производственные задания**. Этот список позволяет руководителям:
- Отслеживать прогресс операций по различным производственным заказам.
- Фильтровать задачи по рабочим центрам или датам.
- Редактировать детали операций, не открывая отдельные производственные заказы.

## Панель загрузки рабочих центров

Панель **Загрузка рабочих центров** — это мощный инструмент для визуального планирования и управления мощностями.

### Доступ к панели
Перейдите в **Производство** → **Рабочие столы** → **Загрузка рабочих центров**.

### Обзор интерфейса

Панель представляет собой матричный вид:
- **Строки**: Список всех настроенных **Рабочих центров**.
- **Столбцы**: Дни выбранного месяца.
- **Заголовок**: Содержит элементы управления для навигации и фильтрации.

#### Элементы управления навигацией
- Используйте поле **Интервал**, чтобы увидеть текущий диапазон дат.
- Нажимайте кнопки `<` и `>`, чтобы перемещаться между месяцами.

#### Визуальные индикаторы
Сетка использует цветовую кодировку для быстрого получения информации:
- **Зеленый фон**: Обозначает текущую дату.
- **Розовый фон**: Обозначает выходные дни (суббота и воскресенье).
- **Синий фон**: Подсвечивает работу, запланированную для **Производственного заказа**, выбранного в данный момент в заголовке панели.
- **Жирные цифры**: Длительность работы для выбранного производственного заказа.
- **Маленькие цифры в скобках**: Общая длительность *всех* производственных заданий для этого рабочего центра в этот день.

### Планирование и редактирование с панели

Панель является интерактивной и позволяет быстро вносить корректировки:

#### Сценарий А: Планирование конкретного Производственного заказа
1. Выберите **Производственный заказ** в заголовке панели.
2. Найдите пересечение нужного **Рабочего центра** и **Даты**.
3. **Нажмите на ячейку**:
   - Введите числовое значение, чтобы установить или обновить **Длительность**.
   - Если для этого производственного заказа в этой ячейке не было производственного задания, оно будет создано автоматически.
   - Чтобы **удалить** производственное задание, введите `0` или очистите значение.

#### Сценарий Б: Общее управление нагрузкой
1. Убедитесь, что в заголовке **не выбран производственный заказ**.
2. **Нажмите на ячейку**, чтобы открыть всплывающее окно со всеми производственными заданиями для этого центра/дня.
3. В этом окне вы можете просматривать, редактировать или удалять существующие производственные задания, а также создавать новые для любого производственного заказа.

### Ограничения и проверки
- **Закрытые заказы**: Вы не можете создавать или изменять производственные задания для производственного заказа, который уже закрыт. Система выведет сообщение: *"Производственный заказ уже закрыт"*.
- **Обязательный рабочий центр**: Каждое производственное задание должно быть закреплено за рабочим центром.
