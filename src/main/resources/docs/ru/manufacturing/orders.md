## Где находится

Откройте раздел **«Производство»** → **«Операции»** → **«Производственные заказы»**.

## Для чего нужен производственный заказ

Производственный заказ — основной документ производства. В нем:

- фиксируется, **что** нужно изготовить (или [разобрать](unbuild.md));
- задается **плановое количество**;
- указывается **[спецификация](bom.md)** (состав изделия), по которой рассчитываются материалы;
- выполняется **проверка наличия** и резервирование материалов;
- фиксируется **фактический выпуск** и **фактическое потребление**;
- задается место хранения готовой продукции при завершении;
- отслеживаются статусы [производственных заданий](work-orders.md) непосредственно в карточке заказа.
- формируются производственные задания по операциям спецификации, включая операции вложенных промежуточных спецификаций, при формировании/пересчёте строк.

## Список производственных заказов

Список предназначен для контроля текущих заказов и быстрого перехода в карточку.

Обычно в списке доступны колонки:

- **Номер**
- **Дата запуска**
- **Номенклатура** (что производится)
- **Тип**
- **Организация** (если используется)
- план: **К изготовлению** и **Единица измерения**
- **Спецификация**
- счетчики строк: сколько строк материалов и сколько строк выпуска

Также в списке, как правило, доступны:

- создание нового заказа;
- редактирование;
- удаление (если не запрещено статусом и правами).

## Карточка производственного заказа

Карточка производственного заказа используется для пошагового ведения процесса.

### Основные поля

В верхней части карточки обычно задаются:

- **Тип** — влияет на поведение (например, на разборку и связанные настройки);
- **Дата запуска** — плановая дата и время запуска;
- **Номенклатура** — изделие, которое производится;
- **Ответственный** — по умолчанию может подставляться текущий пользователь;
- **[Спецификация](bom.md)** — состав изделия;
- **Место хранения материалов** — откуда будут списываться материалы.

### Проверка соответствия номенклатуры спецификации

Если выбрана [спецификация](bom.md), система проверяет, что номенклатура в спецификации совпадает с номенклатурой заказа.

Если номенклатура не совпадает, сохранить заказ нельзя.

### Количество к изготовлению

Заказ предполагает плановое количество **к изготовлению**. По этому количеству система может:

- рассчитать плановые количества материалов;
- рассчитать плановые количества выпуска;
- пропорционально пересчитать фактические значения, если вы вводите выпуск одной цифрой.

## Типовые сценарии

### Создать заказ и подготовить к запуску

1. Создайте новый производственный заказ.
2. Заполните тип, номенклатуру, дату запуска.
3. Выберите [спецификацию](bom.md).
4. Проверьте, что рассчитались строки материалов и выпуска.
   - Одновременно [производственные задания](work-orders.md) формируются по операциям спецификации, включая операции вложенных промежуточных спецификаций.
5. Выполните проверку наличия и резервирование материалов.

### Запустить и завершить

1. Переведите заказ «в работу».
2. Управляйте выполнением производственных заданий (опционально):
   - Перейдите на вкладку **Производственные задания**.
   - Используйте действия **Начать** и **Завершить** для отслеживания прогресса операций.
3. Введите фактический выпуск (при необходимости скорректируйте фактическое потребление).
4. Завершите заказ и укажите место хранения готовой продукции.

### Создать заказы поставщикам на материалы

Из списка производственных заказов можно создать [заказы поставщикам](../purchase/orders.md) на материалы, требуемые для выбранных производственных заказов.

1. В фильтрах задайте **Место хранения списания** — будут обработаны только производственные заказы по этому месту хранения.
2. Выделите производственные заказы, по которым нужно закупить материалы.
3. Выполните действие **Создать заказы**.

Система:

- агрегирует требуемые материалы (учитываются только строки потребления, ещё не привязанные к заказу поставщику) по выбранным производственным заказам;
- группирует номенклатуру по основному [поставщику](../masterdata/partners.md) и создаёт по одному заказу поставщику на каждого поставщика;
- создаёт отдельный заказ поставщику (без поставщика) для номенклатуры, у которой основной поставщик не задан;
- проставляет в созданных заказах выбранное место хранения списания;
- связывает каждую обработанную строку потребления с соответствующей строкой нового заказа поставщику, чтобы сохранить связь между производственным заказом и заказом поставщику;
- открывает каждый созданный заказ поставщику для просмотра.

Если **Место хранения списания** не задано, действие выводит сообщение и не создаёт заказы.

### Производственная потребность в автозаказе поставщику

Когда включено автоматическое заполнение заказов поставщикам, производственная потребность учитывается в расчёте **Автозаказ** в заказе поставщику.

В таблице номенклатуры заказа поставщику система показывает:

- **Ожидает потребления** — количества материалов из производственных заказов, ожидающих выполнения;
- **Потреблено** — количества материалов из производственных заказов в статусе «Завершён» за выбранный период заказа.

Эти количества увеличивают рекомендуемое значение **Автозаказ** вместе с потребностью по отгрузкам, чтобы заказ поставщику мог покрывать и потребности продаж, и потребности производства в материалах.
