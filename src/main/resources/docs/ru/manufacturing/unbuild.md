---
title: Разборка (разукомплектация изделия)
---

Этот раздел описывает логику разборки (разукомплектации), реализованную через **[производственный заказ](orders.md)** специального типа.

## Общая идея

Разборка — это «обратное производство»:

- **исходное изделие** списывается с остатков (как материал);
- **компоненты** из [спецификации](bom.md) приходуются в место хранения готовой продукции (как продукция).

Фактически система оформляет операцию так:

1) списать исходное изделие;
2) оприходовать набор компонентов, рассчитанный по [спецификации](bom.md).

## Что нужно для работы разборки

1. Должен существовать тип производственного заказа с флагом **«Разборка»**.
2. Для разбираемого изделия должна быть задана [спецификация](bom.md) — она определяет, на что разбирается изделие.

Примечание: в начальных данных обычно уже есть тип «Разборка», настроенный как разборка.

## Чем разборка отличается от обычного производства

### Какие строки считаются «материалами», а какие — «продукцией»

При обычном производстве:

- **материалы** = компоненты [спецификации](bom.md);
- **продукция** = готовое изделие (и возможные [отходы](by-products.md)).

При разборке:

- **материалы** = само исходное изделие (и дополнительные позиции, если они указаны как отходы в [спецификации](bom.md));
- **продукция** = компоненты [спецификации](bom.md).

### Плановое количество

Поле **«Произвести»** используется и для разборки, но смысл меняется:

- при разборке это количество исходного изделия, которое будет разобрано (технически — плановый расход исходного изделия).

## Автоматическое формирование строк по спецификации

В заказе доступно действие формирования строк (обычно **«Заполнить по спецификации»**, доступно в статусе «Черновик»).

При разборке система выполняет:

1. Очищает существующие строки материалов и продукции.
2. По компонентам [спецификации](bom.md) создает строки **продукции**:
   - номенклатура = компонент;
   - плановое количество **«Произвести»** рассчитывается пропорционально:
     - количество компонента в спецификации × количество разборки / количество спецификации;
   - если у компонента заполнена **«Доля стоимости»**, она переносится в строку продукции.
3. Создает одну строку **материала** для исходного изделия:
   - номенклатура = изделие заказа;
   - количество **«Израсходовать»** = количество разборки.
4. Если в [спецификации](bom.md) указаны [отходы](by-products.md), они добавляются как дополнительные строки **материалов** (то есть тоже к расходу).

## Резервирование и запуск разборки

Разборка проходит по тем же статусам, что и обычное производство:

1. Подготовьте заказ: тип «Разборка», изделие, [спецификация](bom.md), количество.
2. Проверьте наличие и зарезервируйте материалы.
   - для разборки материалом является исходное изделие;
   - если исходного изделия нет в наличии в месте хранения материалов, заказ не сможет перейти в статус **«В работе»**.
3. Переведите заказ в статус **«В процессе»**.
   - выполните действие **«Произвести»**.
4. Зафиксируйте фактическое выполнение:
   - при разборке фактическое количество **«Произведено»** в шапке — это фактически израсходованное количество исходного изделия;
   - его ввод распределяет фактический выпуск компонентов и расход пропорционально плану.
5. Выполните действие **«Провести»** и укажите поле **«Получатель»** (место хранения, куда поступят компоненты).

## Себестоимость при разборке

Логика оценки та же, что и для производства:

- основа себестоимости — фактический расход (для разборки это расход исходного изделия и дополнительных строк материалов);
- итоговая сумма распределяется по строкам продукции (компонентам) по значениям **«Доля стоимости»**.

Если доли стоимости не заполнены, себестоимость разборки на компоненты не распределяется вовсе — заполните **«Доля стоимости»** в компонентах спецификации (откуда она переносится в строки продукции) или непосредственно в строках продукции.

Подробнее см. [Себестоимость: как рассчитывается](costing.md).

## Типовые ошибки

- **Не формируются строки разборки** — не выбрана спецификация.
- **Не удается зарезервировать** — в месте хранения материалов нет исходного изделия в нужном количестве.
- **Компоненты поступают не в то место хранения** — проверьте, что при проведении заполнено поле **«Получатель»**.
