---
title: OpenAI и GPT-конфигурации
---

## Где находится

Откройте **«Администрирование» → «Приложение» → «Интеграционные данные»** и используйте блок **OpenAI**.

## API-ключ

Заполните **API-ключ** OpenAI, который используется для всех GPT-запросов.

## Конфигурации GPT

Используйте **конфигурации GPT**, чтобы задать повторно используемые параметры запросов:

- **Наименование** — как конфигурация будет отображаться при выборе.
- **Модель** — модель OpenAI для запроса. Если модель не указана ни в запросе, ни в конфигурации, система использует `gpt-5`.
- **Рассуждение** — необязательное значение уровня рассуждения, поддерживаемое выбранной моделью. Оставьте поле пустым, если его не нужно отправлять.
- **Дополнительный промпт** — текст, который добавляется к промпту импорта или запроса, например общие правила формата или внутренние инструкции компании.

Если существует ровно одна конфигурация, она выбирается автоматически для действий импорта, которые создают GPT-запросы. Если конфигураций несколько, система предлагает выбрать одну из них перед отправкой файла.

## GPT-запросы

В том же блоке OpenAI отображается история GPT-запросов. В карточке запроса доступны:

- конфигурация, модель, рассуждение и признак генерации изображений;
- промпт, ответ, статус, ID ответа, время отправки и счетчики токенов;
- входные файлы и выходные файлы, созданные OpenAI;
- отладочные файлы запроса и ответа.

Используйте **Отправить** в карточке запроса для ручной отправки. В запросе сохраняются фактические модель, рассуждение и дополнительный промпт, использованные при отправке.
