---
title: Задания на отбор
---

Задания на отбор (сборку) используются для организации отбора товара в [месте хранения](locations.md) при выполнении [отгрузки](shipments.md).

## Когда создаётся задание на отбор

Здесь есть два связанных признака, и они отвечают за разные вещи:

- **«Отбор»** (на **типе отгрузки**) — включает на карточке отгрузки вкладку **«Отбор»**, где видно резервирование и отобранные количества в разрезе исходного [места хранения](locations.md).
- **«Задание на отбор»** — собственно, создаёт отдельную запись **задания на отбор** для складского оператора. Этот признак есть **и на типе отгрузки, и на самой отгрузке**. Признак на типе только **задаёт значение по умолчанию** для одноимённого признака на отгрузке в момент выбора типа; система проверяет признак **на отгрузке**, когда решает, создавать ли задание. Поэтому изменение признака на типе не влияет на уже созданные отгрузки задним числом, и его можно выключить на конкретной отгрузке, чтобы подавить создание задания.

Задание на отбор создаётся автоматически для отгрузки, удовлетворяющей одновременно двум условиям:

- на самой **отгрузке** включён признак **«Задание на отбор»** (по умолчанию подставляется из типа при выборе типа);
- отгрузка переведена в статус **«В работе»**.

Если у типа отгрузки включён только **«Отбор»**, но **«Задание на отбор»** выключено, вкладка «Отбор» отображается, но отдельная запись задания не формируется.

То есть отбор — это не отдельный пункт меню, а сценарий, который запускается поверх существующей отгрузки.

## Структура

Каждое задание на отбор содержит одну или несколько **строк отбора**. Строка отбора несёт:

- товар и требуемое количество;
- уже отобранное количество и **оставшееся** количество (требуемое минус отобранное);
- сотрудника, выполнившего действие, и время действия;
- опционально — штрихкод-ссылку на товар для быстрого сканирования.

## Типовой процесс

1. Складской оператор открывает форму **«Мобильный отбор»** из группы **«Процессы»** (**«Склад» → «Процессы» → «Мобильный отбор»**).
2. Он выбирает задание, соответствующее обрабатываемой отгрузке.
3. По каждой строке:
   - сканирует штрихкод ячейки/товара (если включено);
   - вводит фактически отобранное количество.
4. Когда всё отобрано, оператор подтверждает задание действием **«Провести»**. Отобранные количества попадают в исходную отгрузку.

Отобранные количества можно вводить и без мобильной формы — прямо на карточке отгрузки.

## Связь с отгрузкой

В карточке отгрузки вкладка **«Отбор»** показывает резервирование и отобранные количества в разрезе исходного [места хранения](locations.md). Вкладка доступна с момента, когда отгрузка отмечена к обработке (статус **«В ожидании»** и все последующие, включая **«В работе»** и **«Выполнен»**), если в типе отгрузки включён отбор. При использовании [партий](lots-and-packages.md) отобранное количество внутри места хранения можно дополнительно детализировать по партиям.
