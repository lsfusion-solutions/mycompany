---
title: Задания на сборку
---

Задания на сборку используются для организации отбора товара в [месте хранения](locations.md) при выполнении [отгрузки](shipments.md).

## Когда создаётся задание на сборку

Задание на сборку создаётся автоматически для отгрузки, удовлетворяющей одновременно двум условиям:

- у **типа отгрузки** включена сборка;
- отгрузка переведена в статус **«Готова»**.

То есть сборка — это не отдельный пункт меню, а сценарий, который запускается поверх существующей отгрузки.

## Структура

Каждое задание на сборку содержит одну или несколько **строк сборки**. Строка сборки несёт:

- товар и требуемое количество;
- уже собранное количество и **оставшееся** количество (требуемое минус собранное);
- сотрудника, выполнившего действие, и время действия;
- опционально — штрихкод-ссылку на товар для быстрого сканирования.

## Типовой процесс

1. Складской оператор открывает форму **«Мобильное задание на сборку»** из группы **«Процессы»**.
2. Он выбирает задание, соответствующее обрабатываемой отгрузке.
3. По каждой строке:
   - сканирует штрихкод ячейки/товара (если включено);
   - вводит фактически отобранное количество.
4. Когда всё отобрано, оператор подтверждает задание. Отобранные количества попадают в исходную отгрузку.

## Связь с отгрузкой

В карточке отгрузки вкладка **«Сборка»** показывает резервирование и собранные количества в разрезе исходного [места хранения](locations.md). Вкладка доступна, пока отгрузка находится в статусе **«Ожидание»** или **«Выполнена»** и в типе отгрузки включена сборка.
