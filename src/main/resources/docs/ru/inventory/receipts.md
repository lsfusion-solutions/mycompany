## Где находится

Откройте раздел **«Склад» → «Операции» → «Приемки»**.

## Назначение

Приемка (поступление) фиксирует приход товара в [место хранения](locations.md).

Документ используется, чтобы:

- зафиксировать планируемое и фактическое количество к приёмке;
- принять товар в конкретное [место хранения](locations.md);
- при необходимости — разнести товар по зонам/ячейкам (адресное хранение);
- сформировать движения в учёте остатков и (при включённом расчёте) себестоимости.

## Список приемок

![Список приемок](images/receipts-list.png)

В списке обычно видны:

- номер;
- планируемая дата и время;
- тип приемки;
- поставщик (если ведётся);
- [место хранения](locations.md);
- примечание;
- количество строк.

Над списком доступны фильтры по **интервалу дат**, **типу**, **месту хранения** и **поставщику**.

### Действия в списке

- **Создать**, **открыть**, **удалить** документы (если разрешено статусом и правами).
- **«Незапланированная приемка»** — создаёт приемку с установленным признаком **«Незапланированная»**; такая приемка пропускает шаг «В работе» и может быть проведена прямо из черновика.
- Групповые действия над **отмеченными** документами: **«В работу»**, **«Провести»**, **«Копировать»** (копирует отмеченные приемки на новую дату) и **«Удалить»**.
- Если используются перемещения с подтверждением приёмки, в списке также есть вкладка **«Подтверждение приемки»** с входящими [перемещениями](transfers.md), ожидающими принятия на вашем месте хранения (см. [Отгрузки](shipments.md#подтверждение-приёмки-в-месте-назначения)).

## Карточка приемки

![Карточка приемки](images/receipt-card.png)

### Шапка документа

В шапке приемки обычно заполняются:

- **Тип** — влияет на нумерацию, [место хранения](locations.md) по умолчанию и ограничения;
- **Планируемая дата** — планируемое время приёмки;
- **Номер** — формируется по нумератору;
- **Поставщик** (если используется);
- **Место хранения** — обязательное поле;
- **Внутренний код поставщика** (например, номер накладной поставщика — если используется);
- **Примечание**.

Дата фактического выполнения (**«Дата исполнения»**) проставляется автоматически при проведении.

Практический совет: сначала выберите тип и [место хранения](locations.md) — после этого удобнее добавлять строки.

### Строки приемки

В строках указываются товары и количество к приёмке.

В типовом наборе полей строки доступны:

- **Товар** — обязательное поле;
- **Единица измерения** — подставляется из номенклатуры;
- **Штрихкод** (если используется);
- **Код** (если используется);
- **Ссылка/артикул** (если используется);
- **Себестоимость** — показывается, если в типе приемки включён признак **«Показывать себестоимость»**; позволяет ввести входную стоимость строки вручную (см. [себестоимость товаров](costing.md));
- колонки упаковок (**«Вид упаковки»**, **«Кол-во мест»**, **«Кол-во в месте»**) — показываются, если в типе приемки включён признак **«Показывать количество мест»** (см. [Кол-во мест](product-sku.md#альтернатива-учет-в-упаковках-местах-в-документах)).

#### Поле «Планируемое кол-во»

Для приемок, которые не выполняются немедленно, в строке используется поле **«Планируемое кол-во»**:

- это планируемое количество к приёмке по строке;
- поле может подсвечиваться в черновике, чтобы обратить внимание на необходимость заполнения.

Ограничение:

- значение должно быть в диапазоне от `0` до **максимального количества**, указанного в типе приемки;
- если превышено, документ не сохранится.

#### Ограничение «одна строка на один товар»

Для некоторых типов приемок может быть включено правило:

- один и тот же товар нельзя добавить двумя строками;
- при попытке добавить дубль система выдаст ошибку.

### Вкладка «Поиск» и ввод по штрихкоду

Для быстрого заполнения строк на карточке есть вкладка **«Поиск»**:

- товары ищутся по категориям и атрибутам, рядом показываются актуальные количества **остаток / ожидается / доступно**;
- планируемое или принятое количество можно вводить прямо в результатах поиска — строка документа создаётся автоматически;
- на вкладке строк есть поле ввода штрихкода: сканирование штрихкода товара добавляет строку (или увеличивает количество существующей).

### Прочие вкладки

Кроме строк, на карточке есть вкладки **«История»** (журнал изменений: кто и когда менял статус, даты и строки), **«Комментарии»** (комментарии с упоминаниями) и **«Файлы»** (вложения). При включённых соответствующих механизмах появляются вкладки **«Партии»**, **«Упаковки»** и панель **«Размещение»** (см. ниже), а при использовании бухгалтерского модуля — вкладка **«Проводки»** с проводками документа.

## Статусы и этапы

Ниже приведён **точный набор статусов**, который следует из исходного кода.

1. **Черновик** — ввод данных.
2. **В работе** — документ отмечен к выполнению.
3. **Выполнен** — факт приёмки подтверждён, фиксируется дата выполнения.
4. **Размещение** — размещение по дочерним местам хранения.
   - статус доступен **только если** в типе приемки включён признак размещения;
   - в размещении контролируется, что выбранное [место хранения](locations.md) является дочерним от места хранения, указанного в документе, и что суммарно размещено не больше, чем принято.
5. **Отменен** — документ отменён.

### Действия смены статуса

В карточке приемки доступны действия, переводящие документ между статусами:

- **«В работу»** — перевод из **«Черновик»** в **«В работе»** (доступно также как групповое действие в списке приемок). Для типов с признаком **«Увеличивать доступное количество»** приемка в статусе «В работе» увеличивает *ожидаемое* количество в регистре резервов — приходящий товар уже учитывается в доступном остатке.
- **«Провести»** — подтверждение факта и перевод в **«Выполнен»**; дата выполнения проставляется автоматически. Кнопка показывается, когда документ находится в **«В работе»** (обычный поток: «Черновик» → «В работу» → «В работе» → «Провести»). Для **незапланированных** приемок — у которых признак **«Незапланированная»** установлен на самом документе (а также создаваемых действием **«Незапланированная приемка»** из списка) — та же кнопка показывается и прямо из **«Черновик»**, поскольку такие приемки пропускают шаг «В работе». В списке доступно одноимённое групповое действие. Вспомогательная команда **«Заполнить кол-ва (выполненные)»** одной кнопкой копирует «Планируемое кол-во» в фактически принятое по всем строкам. Если принятое количество отличается от планируемого, система предупредит о расхождении — если только в типе приемки не включён признак **«Не проверять принятое количество»**.
- **«Размещение»** — для типов приемок, поддерживающих размещение, переводит документ из **«Выполнен»** в **«Размещение»** после того, как заполнены строки размещения.
- **«Отменить»** — перевод документа в **«Отменен»**.
- **«Копировать»** — создаёт новую приемку в статусе «Черновик» с такой же шапкой и строками.

## Размещение (адресное хранение)

Если используется адресное хранение, после приёмки выполняется размещение по ячейкам:

- на панели **«Размещение»** по каждой строке указывается, в какие дочерние [места хранения](locations.md) раскладывается принятое количество;
- вспомогательное действие **«Разместить (+)»** заполняет оставшееся количество;
- при использовании [партий](lots-and-packages.md) размещение можно детализировать по партиям (действие **«Выбрать»** предзаполняет партии из принятых количеств).

Рекомендация: сначала завершайте приёмку (зафиксируйте факт), затем выполняйте размещение — так проще контролировать расхождения.

## Партии в приемках

Когда для товара включён [партионный учет](lots-and-packages.md), на карточке приемки появляется вкладка **«Партии»**:

- по каждой строке указываются партии и их количества;
- действие **«Сгенерировать»** создаёт партии автоматически по нумератору и префиксу, настроенным на категории товара. Для товаров с включёнными **«Серийными номерами»** генерируется по одной партии на единицу (каждая количеством 1); иначе одна партия на весь остаток строки;
- сканирование штрихкода партии привязывает партию к строке; для неизвестного штрихкода партия создаётся автоматически;
- по строкам можно печатать этикетки партий (см. [партии и упаковки](lots-and-packages.md)).

## Упаковки в приемках

Если используется справочник [упаковок](lots-and-packages.md#упаковки), на карточке появляется вкладка **«Упаковки»**, где существующие упаковки привязываются к приемке (действия **«Добавить»** / **«Удалить»**). Строки упаковки показываются для справки и прослеживаемости; остатки по-прежнему проводятся из собственных количеств строк приемки.

## Возвраты поставщику

Если тип приемки связан с типом возвратной [отгрузки](shipments.md) (секция **«Возврат»** в настройках типа), на активных приемках доступно действие **«Возврат»**:

- оно открывает новую возвратную отгрузку, предзаполненную местом хранения и товарами приемки;
- колонка **«Возвращено»** в строках приемки показывает уже возвращённое количество, а диалог **«Возвраты»** — связанные возвратные отгрузки;
- при включённом признаке **«Проверять возвращенное количество»** система запретит вернуть больше, чем принято.

## Создание приемок из заказов поставщику

Если используется модуль «Закупки» и тип заказа поставщику связан с типом приемки, при проведении заказа система автоматически создаёт связанную приемку в статусе **«В работе»** со строками заказа. В списке приемок тогда виден исходный **заказ**, а в карточке заказа — его **приемки**. Ограничениями можно запретить закрывать заказы с активными или не полностью принятыми приемками.

## Печать

- **«Печать»** — печать приемки по настраиваемому шаблону (шаблоны назначаются на тип приемки в настройках).
- **«Этикетки»** — печать этикеток товаров по принятым строкам (по одной этикетке на принятую единицу).

## Импорт начальных остатков

Для запуска системы начальные остатки можно загрузить из Excel (действие **«Импорт остатков»** на форме миграции данных): в файле указываются место хранения, товар и количество (опционально — себестоимость), а система создаёт начальные приемки. Обратное действие **«Экспорт остатков»** выгружает текущие остатки в Excel.

## Типовые проблемы

- **Не удаётся сохранить строку** — значение «Планируемое кол-во» выходит за пределы, заданные в типе приемки.
- **Не удаётся добавить товар второй строкой** — для типа приемки включено правило «одна строка на один товар».
- **Не удаётся завершить** — не заполнено [место хранения](locations.md) или есть строки без количества.
- **Не совпадают фактические количества** — проверьте ввод по строкам и единицы измерения.
- **При проведении система предупреждает о расхождении количеств** — принятое количество отличается от планируемого; исправьте количества или включите в типе признак **«Не проверять принятое количество»**.
