## Где находится

Откройте **«Склад» → «Настройка» → «Настройки»**.

![Форма «Настройки» модуля «Склад»](images/settings.png)

## Что обычно настраивается

- типы [приемок](receipts.md) и [отгрузок](shipments.md) (их нумерация, места хранения по умолчанию, максимальное количество и другие признаки);
- использование [перемещений](transfers.md) ([отгрузка](shipments.md) с признаком **«Перемещение»**);
- типы [списаний](scrap.md);
- включён ли **[партионный учет](lots-and-packages.md)** (глобальный признак), а также настройки партий/серийных номеров по категориям и товарам;
- сквозные признаки, такие как **«Запретить несколько корневых мест хранений»**;
- поместные признаки **«Запрет отрицательных остатков»** и **«Запретить резервирование сверх доступного»** задаются в карточке самого [места хранения](locations.md) (см. про ограничения регистров ниже).

## Типы приемок

В настройках ведётся справочник **типов приемок**. Тип приемки влияет на то, как пользователь работает с документом.

В типе приемки задаются:

- **Нумератор** — правило формирования номера;
- **Место хранения** (по умолчанию) — какое [место хранения](locations.md) подставлять в новые документы;
- **Максимальное количество** — верхняя граница для поля «Планируемое кол-во» в строках;
- **«Только одна строка для одной номенклатуры»** — запрет добавлять один и тот же товар двумя строками;
- **«Не проверять принятое количество»** — позволяет завершать приемку, у которой принятое количество отличается от планируемого, без предупреждения;
- **«Размещение»** — включает этап размещения по ячейкам (см. [приемки](receipts.md));
- **«Показывать себестоимость»** — добавляет в строки приемки колонку **«Себестоимость»** для ручного ввода входной стоимости (см. [себестоимость товаров](costing.md));
- **«Показывать количество мест»** — добавляет в строки колонки упаковочных единиц (см. [Кол-во мест](product-sku.md#альтернатива-учет-в-упаковках-местах-в-документах));
- **«Увеличивать доступное количество»** — приемки этого типа в статусе **«В работе»** увеличивают *ожидаемое* количество в регистре резервов, то есть товар учитывается в доступном остатке ещё до фактического прихода;
- секция **«Возврат»** — тип [отгрузки](shipments.md), используемый для возврата товара поставщику, и признак **«Проверять возвращенное количество»**, запрещающий возвращать больше, чем принято.

Если в системе создан ровно один тип приемки, он подставляется в новые документы автоматически.

## Типы отгрузок

В настройках ведётся справочник **типов отгрузок**.

В типе отгрузки задаются:

- **Нумератор**;
- **Место хранения‑источник** (по умолчанию);
- **Место хранения‑назначение** (по умолчанию; актуально для перемещений);
- **Признак «Перемещение»** — включает режим «место хранения‑источник → место хранения‑назначение»;
- **Максимальное количество** — верхняя граница для поля «Планируемое кол-во» в строках;
- **«Только одна строка для одной номенклатуры»** — запрет добавлять один и тот же товар двумя строками;
- **«Не проверять отгруженное количество»** — позволяет завершать отгрузку, у которой отгруженное количество отличается от планируемого, без предупреждения;
- **«Отбор»** — показывает на карточке отгрузки вкладку **«Отбор»** (доступность/резерв по местам хранения, см. [задания на отбор](picking.md));
- **«Задание на отбор»** — задаёт значение по умолчанию для одноимённого признака отгрузки, который включает создание [заданий на отбор](picking.md);
- **«Показывать количество мест»** — добавляет в строки колонки упаковочных единиц;
- секция **«Возврат»** — тип [приемки](receipts.md), используемый для возвратов от покупателя, и признак **«Проверять возвращенное количество»**.

Проверка:

- для перемещений место хранения‑источник и место хранения‑назначение по умолчанию не могут совпадать.

## Типы списаний

Справочник **типов [списаний](scrap.md)** классифицирует причины списания (например, «Порча», «Потери», «Истечение срока»). У типа списания есть собственный **нумератор** и **место хранения** по умолчанию.

## Типы инвентаризаций

[Инвентаризации](adjustments.md) используют фиксированный набор типов, определяющий способ заполнения строк:

- **«Все»** — пересчёт всех товаров с остатком в месте хранения;
- **«По категории»** — пересчёт только товаров выбранной категории;
- **«Вручную»** — строки вводятся вручную (или через вкладку поиска/сканирование штрихкодов).

## Ограничения регистров (по месту хранения)

У каждого [места хранения](locations.md) можно включить два опциональных ограничения:

- **«Запрет отрицательных остатков»** — система не разрешит операции, которые загоняют физический остаток товара в этом месте хранения в минус.
- **«Запретить резервирование сверх доступного»** — система не разрешит, чтобы доступный остаток (*остаток − резерв + ожидается*) в этом месте хранения ушёл в минус.

Оба признака наследуются дочерними местами хранения. Когда признак включён, соответствующие проводки в регистр остатков или регистр резервов блокируются с пояснительным сообщением.

## Рекомендованный порядок настройки

1. Настройте [места хранения](locations.md) (включая признак **«Расчет себестоимости»** и ограничения регистров).
2. Настройте типы документов ([приемки](receipts.md)/[отгрузки](shipments.md)/[перемещения](transfers.md)/[списания](scrap.md)).
3. При необходимости включите глобально [партионный учет](lots-and-packages.md) и настройте опции партий/серийных номеров по категориям и товарам.
4. Просмотрите [отчёты и регистры](reports-and-ledgers.md) и права доступа (включая [доступ к местам хранения по сотрудникам](locations.md#доступ-по-сотрудникам)).
