Страница описывает базовые настройки, влияющие на работу **[кассы](pos.md)** и **[POS](pos.md)**.

## Где находится

Настройки обычно находятся в разделе **«Розница» → «Настройка» → «Настройки»**.

В большинстве конфигураций справочники **«Кассы ККМ»** и **«Дисконтные карты»** доступны прямо из этого раздела; **методы оплаты** отдельным справочником не являются — они ведутся на самой форме **«Настройки»** (см. ниже).

![Форма «Настройки» модуля «Розница»](images/settings.png)

## Кассы

Касса — это рабочее место, с которого оформляются продажи и возвраты.

Где искать: обычно **«Розница» → «Настройка» → «Кассы ККМ»**.

Обычно настраиваются:

- **имя** и **код** кассы;
- **организация**;
- **тип реализации** для чеков, создаваемых на этой кассе (обязательно) — он же определяет соответствующий тип возврата;
- **место хранения** (необязательно) — подставляется в продажи и возвраты этой кассы, определяет доступность остатков и фильтр чеков **«По месту хранения»**; место хранения для внутреннего использования выбрать нельзя;
- привязка кассы к одному или нескольким **компьютерам** — если у компьютера есть хотя бы одна привязанная касса, на нём предлагаются только привязанные кассы; компьютер без привязок видит все кассы. В любом случае из выбора скрываются кассы, чьё место хранения недоступно текущему пользователю (кассы без места хранения остаются видимыми);
- **счета по методам оплаты** — на отдельной вкладке кассы для каждого **[метода оплаты](payments.md)** можно указать **счёт**, на который относятся принятые этим методом платежи, и пометить метод как **отключённый**, чтобы он не предлагался в диалогах оплаты продажи и возврата на этой кассе.

![Справочник касс ККМ](images/cash-registers.png)

> **Наличный счёт.** Чтобы на экране POS работали операции **«Внесение денег»** и **«Выемка»** (а на вкладке «Смена» отображался остаток в поле **«Наличных в кассе»**), для метода оплаты **«Наличные»** у кассы должен быть указан **счёт**. Пока наличный счёт не задан, кнопки внесения и выемки денег на экране POS остаются **недоступными** (неактивны). Кроме того, для самих операций внесения/выемки на вкладке **«Основные»** формы настроек должны быть заданы **«Тип внесения денег»** и **«Тип изъятия денег»**.

> **Счёт центральной кассы.** Поле **«Касса»** в шапке карточки кассы — это **счёт центральной кассы**, контрагентский счёт для операций **«Внесение денег»** и **«Выемка»**: при внесении деньги переводятся со счёта центральной кассы в **денежный ящик ККМ** (счёт метода оплаты **«Наличные»**), при выемке — обратно. Поэтому счёт в поле **«Касса»** должен **отличаться** от счёта, указанного для метода оплаты **«Наличные»**. Если это один и тот же счёт, внесение/выемка проводит обе стороны перевода по одному счёту, сумма обнуляется, и остаток в поле **«Наличных в кассе»** не меняется. Система это контролирует — кассу с одинаковым счётом в обоих местах сохранить не получится. Кроме того, счёт центральной кассы и каждый счёт метода оплаты должны принадлежать **организации** кассы, иначе кассу сохранить нельзя.

## Смены

**[Смены](sessions.md)** нумеруются автоматически. Нумератор смен выбирается на вкладке **«Основные»** формы настроек.

## Методы оплаты

Список методов оплаты ведётся на вкладке **«Метод оплаты»** формы настроек (см. также: **[Оплата в рознице](payments.md)**). Для каждого метода указываются:

- **имя** и **код**;
- признак **«Наличные»** — отмечает метод как наличный (наличным может быть только один метод; он используется для расчёта сдачи);
- **«Тип входящего платежа»** и **«Тип возвратного платежа»** — типы платежей, которые используются при приёме метода в продаже и при выдаче средств в возврате.

## Дисконтные карты

**[Дисконтные карты](discount-cards.md)** нумеруются автоматически; нумератор дисконтных карт выбирается на вкладке **«Основные»** формы настроек. Сам список карт находится в **«Розница» → «Настройка» → «Дисконтные карты»**.

## Маркированные товары

Если в вашей конфигурации используется работа с **[маркированными товарами](marking.md)**:

- в настройках интеграции **Честный Знак** должны быть заполнены токен и (при необходимости) параметры Sandbox;
- для оффлайн-сценария должен быть настроен локальный модуль ЧЗ (ЛМ ЧЗ) и выполнена его инициализация;
- по категориям может задаваться обязательность сканирования;
- может настраиваться режим проверки кодов (онлайн/оффлайн);
- для оффлайн-проверки может потребоваться указать локальный модуль для кассы.

Подробный пользовательский порядок: [CRPT (Честный Знак) и маркированные товары](marking.md).
