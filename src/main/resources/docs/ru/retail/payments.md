title: Оплата в рознице

Страница описывает оформление оплаты в **[POS](pos.md)**: методы оплаты, распределение сумм и контроль сдачи.

## Методы оплаты

Метод оплаты — это настроенный вариант приёма/выдачи денег (например, **наличные**, **банковская карта**). Список методов оплаты и их доступность могут зависеть от **[кассы](settings.md)**.

Настройка методов оплаты: см. [Настройки](settings.md).

## Как принять оплату

1. В чеке перейдите к оплате.
2. Проверьте сумму **к оплате**.
3. Укажите суммы по одному или нескольким методам оплаты.
4. Подтвердите оплату.

> Если суммы не вводить вообще, при подтверждении вся сумма **«К оплате»** будет отнесена на выделенный метод оплаты.

### Сдача

Если наличными внесено больше суммы к оплате, система рассчитает **сдачу**. Внести больше суммы к оплате можно только наличными.

### Контроль ошибок ввода

При вводе сумм действуют проверки:

- нельзя подтвердить оплату, если внесённой суммы недостаточно;
- нельзя «переплатить» безналичными методами (например, банковской картой) сверх суммы к оплате;
- при комбинированной оплате учитывается сумма по всем методам.

## Оплата возврата

При возврате оформляется выдача средств покупателю. Правила распределения сумм фиксированы: по каждому методу оплаты нельзя вернуть больше, чем было оплачено этим методом в исходном чеке, а общая сумма выдачи должна быть равна сумме возврата. Этот лимит по методу проверяется **на каждый возврат** относительно исходного чека — ранее сделанные возвраты по тому же чеку не вычитаются, то есть это не накопительный лимит по нескольким возвратам. Настройки не меняют эти лимиты сумм, но определяют состав доступных методов оплаты и тип платежа / счёт кассы, на который относится каждая выдача.

Подробности по шагам и типовым ограничениям: [Возвраты](returns.md).
