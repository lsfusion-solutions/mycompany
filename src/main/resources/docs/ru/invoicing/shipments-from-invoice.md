---
title: Создание отгрузки на основании реализации
---

В системе предусмотрен сценарий, когда [отгрузка](../inventory/shipments.md) создаётся **из [реализации](invoices.md)**.

Это удобно, если реализация является «основным» документом, по которому дальше формируется складская отгрузка.

## Когда доступно создание

Кнопка **«Создать отгрузку»** в карточке реализации доступна, если:

- реализация **активна** — то есть в статусе **«К оплате»** или позже и не отменена (в статусе **«Черновик»** кнопка не показывается);
- для [типа реализации](settings.md) задан тип отгрузки (см. [Настройки склада](../inventory/settings.md));
- в реализации есть позиции, которые нужно отгрузить (количество «к отгрузке» больше нуля).

Если по реализации уже создана отгрузка на все позиции, кнопка не отображается.

Замечание о количествах: количество «к отгрузке» выражается в складских единицах. Когда единица продажи товара отличается от его складской единицы, количество в отгрузке равно количеству в реализации, умноженному на коэффициент SKU товара, поэтому количества в реализации и отгрузке могут закономерно отличаться.

## Что делает система при создании

При создании отгрузки из реализации система:

1. Создаёт новый документ отгрузки.
2. Переносит основные реквизиты из реализации:
   - [контрагента](../masterdata/partners.md);
   - [подразделение](../masterdata/departments.md) (если используется);
   - [место хранения](../inventory/locations.md);
   - адрес доставки.
3. По строкам реализации создаёт строки отгрузки:
   - в отгрузку попадают только товарные позиции;
   - количество в отгрузке равно количеству «к отгрузке» по строке реализации;
   - строка отгрузки связывается со строкой реализации.
4. Открывает созданную отгрузку для дальнейшей работы.

## Автоматическое создание отгрузки

Для [типа реализации](settings.md) может быть включена настройка **«Автоматически создавать отгрузку»**.

Если она включена, отгрузка создаётся автоматически **при переходе реализации в статус «К оплате»**. Добавление товарных строк в реализацию, которая уже находится в статусе «К оплате», также запускает создание; добавление строк в реализацию-черновик — нет (реализация должна уже быть в статусе «К оплате»).

## Синхронизация изменений

Когда отгрузка создана автоматически (то есть у типа установлен признак **«Автоматически создавать отгрузку»**) и реализация связана с единственной отгрузкой, система поддерживает их согласованность:

- если реализация отменена — отгрузка автоматически отменяется;
- если изменён покупатель — он обновляется в отгрузке;
- если изменилось количество в строке реализации — обновляется отгружаемое количество в связанной строке отгрузки;
- если изменился товар в строке реализации — обновляется товар в связанной строке отгрузки.

Эти правила применяются только к автоматически созданным отгрузкам; отгрузка, созданная вручную действием **«Создать отгрузку»**, повторно не синхронизируется.

Практический смысл: автоматически созданная отгрузка остаётся согласованной с реализацией, пока вы работаете с документами.

## Типовой сценарий

1. Откройте реализацию.
2. Нажмите **«Создать отгрузку»**.
3. Проверьте место хранения и адрес доставки.
4. Проверьте количество по строкам (что отгружается именно то, что нужно).
5. Проведите/подтвердите отгрузку.

## Плановая и немедленная отгрузка

Отгрузка, созданная из реализации, может быть либо **плановой**, либо **немедленной**. Режим определяется признаком **«Создать плановую отгрузку»** на [типе реализации](settings.md):

- если признак **включён**, созданная отгрузка — это обычная плановая [отгрузка](../inventory/shipments.md): количества «к отгрузке» становятся **исходной потребностью** строк отгрузки, и она проходит привычный жизненный цикл Ожидание / Готова / Выполнена;
- если признак **выключен**, созданная отгрузка отмечается как немедленная: количества «к отгрузке» сразу записываются в строки как **выполненные**, а отгрузка переводится в **«Выполнена»** в момент создания. В этом режиме отгрузка фактически является пассивной записью о том, что реализация уже задекларировала как отгруженное.

## Обратное направление: реализация из отгрузок

Поддерживается и обратный сценарий — [реализация](invoices.md) может быть создана из одной или нескольких уже существующих отгрузок. Действие располагается в списке отгрузок и обычно называется **«Создать реализацию»**:

1. Откройте список отгрузок.
2. Выделите одну или несколько отгрузок одного покупателя.
3. Запустите **«Создать реализацию»** — система создаст новую реализацию-черновик. Она агрегирует выполненные количества выбранных отгрузок **по товару** и включает только те строки отгрузок, которые ещё не привязаны к строке реализации; шапка реализации (покупатель, подразделение, место хранения) берётся из одной из выбранных отгрузок, поэтому лучше выбирать отгрузки с одинаковыми значениями этих полей.

Это удобно, когда склад сначала оформляет отгрузку, а реализация выставляется потом.

См. также: [Расчёты → Реализации](invoices.md); [Склад → Отгрузки](../inventory/shipments.md).
