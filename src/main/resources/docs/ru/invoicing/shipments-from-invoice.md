---
title: Создание отгрузки на основании реализации
---

В системе предусмотрен сценарий, когда [отгрузка](../inventory/shipments.md) создаётся **из [реализации](invoices.md)**.

Это удобно, если реализация является «основным» документом, по которому дальше формируется складская отгрузка.

## Когда доступно создание

Кнопка создания отгрузки в карточке реализации обычно доступна, если:

- реализация активна (не отменена);
- для [типа реализации](settings.md) задан тип отгрузки (см. [Настройки склада](../inventory/settings.md));
- в реализации есть позиции, которые нужно отгрузить (есть «к отгрузке» больше нуля).

Если по реализации уже создана отгрузка на все позиции, кнопка не отображается.

## Что делает система при создании

При создании отгрузки из реализации система:

1. Создаёт новый документ отгрузки.
2. Переносит основные реквизиты из реализации:
   - [контрагента](../masterdata/partners.md);
   - [подразделение](../masterdata/departments.md) (если используется);
   - [место хранения](../inventory/locations.md);
   - адрес доставки.
3. По строкам реализации создаёт строки отгрузки:
   - в отгрузку попадают только товарные позиции;
   - количество в отгрузке равно количеству «к отгрузке» по строке реализации;
   - строка отгрузки связывается со строкой реализации.
4. Открывает созданную отгрузку для дальнейшей работы.

## Автоматическое создание отгрузки

Для [типа реализации](settings.md) может быть включена настройка **автоматического создания отгрузки**.

Если она включена, то при переходе реализации в состояние готовности (или при появлении товарных строк) система автоматически создаёт отгрузку.

## Синхронизация изменений

Если отгрузка создана автоматически, система поддерживает синхронизацию между реализацией и отгрузкой в ряде случаев:

- если реализация отменена и по ней была одна отгрузка — отгрузка автоматически отменяется;
- если изменился контрагент и по реализации одна отгрузка — контрагент в отгрузке обновляется;
- если изменились товар или количество в строке реализации — количество и товар в строке отгрузки обновляются.

Практический смысл: отгрузка остаётся согласованной с реализацией, пока вы работаете с документами.

## Типовой сценарий

1. Откройте реализацию.
2. Нажмите «Создать отгрузку».
3. Проверьте место хранения и адрес доставки.
4. Проверьте количество по строкам (что отгружается именно то, что нужно).
5. Проведите/подтвердите отгрузку.

## Плановая и немедленная отгрузка

Отгрузка, созданная из реализации, может быть либо **плановой**, либо **немедленной**. Режим определяется признаком **«Плановая отгрузка»** на [типе реализации](settings.md):

- если признак **включён**, созданная отгрузка — это обычная плановая [отгрузка](../inventory/shipments.md), которая проходит привычный жизненный цикл Ожидание / Готова / Выполнена;
- если признак **выключен**, созданная отгрузка отмечается как немедленная и автоматически переводится в **«Выполнена»** в момент создания. В этом режиме отгрузка фактически является пассивной записью о том, что реализация уже задекларировала как отгруженное.

## Обратное направление: реализация из отгрузок

Поддерживается и обратный сценарий — [реализация](invoices.md) может быть создана из одной или нескольких уже существующих отгрузок. Действие располагается в списке отгрузок и обычно называется **«Создать реализацию»**:

1. Откройте список отгрузок.
2. Выделите одну или несколько отгрузок одного покупателя, ещё не привязанных к реализации.
3. Запустите **«Создать реализацию»** — система создаст новую реализацию в статусе «Черновик», строки которой ссылаются на строки выбранных отгрузок.

Это удобно, когда склад сначала оформляет отгрузку, а реализация выставляется потом.

См. также: [Расчёты → Реализации](invoices.md); [Склад → Отгрузки](../inventory/shipments.md).
