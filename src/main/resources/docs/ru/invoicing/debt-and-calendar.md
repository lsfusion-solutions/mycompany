## Задолженность

Задолженность рассчитывается как разница:

- сумма документов ([поступления](bills.md), [реализации](invoices.md), [возвраты и корректировки](refunds-and-corrections.md));
- минус сумма привязанных [платежей](payments.md).

Задолженность может считаться:

- по [контрагенту](../masterdata/partners.md);
- по [договору](../masterdata/contracts.md);
- по конкретному документу.

Что важно понимать:

- задолженность меняется **только по тем платежам, которые привязаны/распределены** к документам;
- если документ отменен, он обычно не участвует в расчете;
- при частичной оплате задолженность уменьшается на сумму распределения.

## Как погашается задолженность

1. Создайте [платёж](payments.md).
2. Свяжите платёж с документом (или распределите по нескольким документам).
3. После проведения платежа задолженность уменьшается.

Если платеж распределен на несколько документов — задолженность уменьшится по каждому из них на соответствующую сумму.

## Календарь платежей

Календарь платежей используется для планирования:

- ожидаемых поступлений;
- планируемых выплат.

Обычно календарь строится на основании:

- сроков оплаты в документах;
- условий оплаты;
- текущей задолженности.

### Типовые сценарии использования

1. **Планирование поступлений**: увидеть, какие суммы ожидаются на ближайшую неделю/месяц по условиям оплаты.
2. **Контроль просрочки**: отфильтровать документы с плановой датой оплаты в прошлом.
3. **Планирование выплат**: по [исходящим платежам](outgoing-payments.md) (если ведутся плановые выплаты).

### На что смотреть, если календарь «пустой» или даты неверные

Проверьте:

- заполнены ли в документах **условия оплаты/срок оплаты**;
- включены ли настройки календаря и правила расчета плановой даты;
- не исключены ли документы статусом (например, отменено).

Подробнее про параметры: [Настройки и справочники](settings.md).
