## Где находится

Откройте **«Расчёты» → «Настройка» → «Настройки»**.

![Форма «Настройки» модуля «Расчёты»](images/settings.png)

## Форма «Настройки»

Сама форма **«Настройки»** (**«Настройка» → «Настройки»**) содержит:

- справочники **типов документов** — **типы поступлений**, **типы реализаций**, **типы входящих платежей**, **типы исходящих платежей** — у каждого свой нумератор, контрагент по умолчанию, признак **«Цена включает налоги»**, поведение оплаты/отгрузки и (для возвратов) признак **Return** и связанный с ним **Return type**, описанные в разделе [Возвраты и корректы](refunds-and-corrections.md);
- справочник **групп налогов**;
- переключатель **«Не приводить долг к валюте по умолчанию»** — если он включён, показатели задолженности хранятся в собственной валюте каждого документа, а не пересчитываются в валюту по умолчанию.

Каждый тип документа может задавать **валюту по умолчанию**, которая подставляется в его документы. Это отдельно от общесистемной **валюты по умолчанию**, в которую пересчитываются показатели задолженности (если не включён переключатель выше).

## Справочники раздела «Настройка» (отдельные пункты навигатора)

Помимо формы «Настройки», группа **«Настройка»** содержит следующие справочники как отдельные пункты меню:

- **[«Налоги»](taxes.md)** — налоги и налоговые ставки;
- **«Условия оплаты»** — см. ниже;
- **«Банки»** — справочник банков;
- **«Счета»** — единый справочник, содержащий и **банковские счета**, и **кассы** (добавляются соответствующими кнопками);
- **«Статьи ДДС»** (аналитические счета) — используются для классификации платежей; разрешаются по типу платежа.

Типы платежей дополнительно несут признак **«Внутренний платеж»** и настройку **допустимых типов счетов** (касса / банк, требуется минимум один).

[Базы распределения затрат](bill-cost.md) для распределения стоимости услуг задаются на товарах-услугах, а шаблоны печати настраиваются по типам документов (см. [Печать и отчётность](reports-and-printing.md)).

## Условия оплаты

Условия оплаты несут количество **«Дни»**; будучи назначенными контрагенту (отдельно для продаж и закупок) и скопированными в документы, они определяют:

- расчёт даты **«Оплатить до»**;
- построение [платежного календаря](debt-and-calendar.md);
- контроль просрочки.

## Импорт поступлений по файлу

Если в конфигурации используется распознавание поступлений по файлу, заранее подготовьте две группы настроек:

- в общих параметрах интеграции OpenAI заполните API-ключ. Если нужно управлять моделью, рассуждением или общими дополнительными инструкциями, создайте конфигурации GPT (см. [OpenAI и GPT-конфигурации](../administration/openai.md)). Если модель не указана ни в запросе, ни в конфигурации, используется `gpt-5`;
- в карточке **типа поступления** заполните **промпт** для распознавания. При первом заполнении удобно подставить текст **по умолчанию**, а затем при необходимости уточнить его под ваши документы.

Если доступно несколько конфигураций GPT, при импорте система предложит выбрать конфигурацию перед отправкой файла. Если конфигурация одна, она подставляется автоматически.

Также перед запуском распознавания проверьте справочники:

- поставщики и [номенклатура](../masterdata/items.md) должны быть заведены заранее и быть активными;
- [валюты](../masterdata/currencies.md) и [налоги](taxes.md) должны уже существовать в системе.

Кнопка импорта появляется в карточке поступления только для тех типов, у которых заполнен промпт.
